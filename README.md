# NTIT MAX Bot

Бот для MAX, который:
- принимает обращения от пользователя и создает задачу в Pyrus;
- проверяет ИНН в форме контрагентов Pyrus;
- показывает название найденной компании или сообщает, что компания не найдена.

## Что умеет сейчас

- Диалог создания заявки с шагами: ИНН -> ФИО -> телефон -> имя ПК -> тема -> проблема.
- Проверка ИНН:
  - валидный формат (10 или 12 цифр);
  - поиск компании в Pyrus по ИНН;
  - если найдено, бот показывает название компании;
  - если не найдено, просит ввести ИНН снова.
- Отправка финальной заявки в Pyrus после подтверждения.

## Структура проекта

- `main.py` - точка входа.
- `app/bot.py` - запуск MAX-бота и регистрация роутеров.
- `app/handlers/` - обработчики сообщений и callback-событий.
- `app/pyrus/client.py` - HTTP-клиент Pyrus (авторизация, запросы, создание задачи).
- `app/pyrus/service.py` - бизнес-логика Pyrus (поиск контрагента по ИНН, темы и т.д.).
- `app/data/sqlite.py` - локальная SQLite-БД (связки пользователей, задачи, оценки).
- `app/keyboards.py` - inline-клавиатуры.
- `app/states.py` - FSM-состояния.

## Настройка окружения

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` на основе примера:

```bash
copy .env.example .env
```

4. Заполните переменные в `.env`:

- `MAX_BOT_TOKEN` - токен бота MAX.
- `PYRUS_BASE_URL` - URL API Pyrus (обычно `https://api.pyrus.com/v4`).
- `PYRUS_LOGIN` - логин Pyrus.
- `PYRUS_SECURITY_KEY` - security key Pyrus.
- `PYRUS_PERSON_ID` - person_id пользователя в Pyrus.
- `PYRUS_TASK_FORM_ID` - ID формы, в которую создаются заявки.
- `PYRUS_DEFAULT_PARTICIPANT_ID` - ID ответственного (опционально).
- `PYRUS_DEFAULT_PRIORITY_ITEM_ID` - item_id приоритета по умолчанию.
- `BOT_DB_PATH` - путь к SQLite-файлу (по умолчанию `data/bot.db`).

## Запуск

```bash
python main.py
```

## Важно про форму контрагентов

В `app/pyrus/service.py` сейчас зашиты ID для поиска контрагента:
- `CONTRACTORS_FORM_ID = 2306222`
- `CONTRACTOR_INN_FIELD_ID = 5`
- `CONTRACTOR_NAME_FIELD_ID = 10`

Если в вашем Pyrus другие ID формы/полей, поменяйте их.

## Что хранится в SQLite

- `user_links` - связка пользователя MAX и данных Pyrus/контрагента.
- `tickets` - созданные заявки, включая статусы и отдельный признак закрытия (`resolved_at`).
- `ticket_ratings` - оценки инженера (1..5) и комментарий пользователя по задаче.
